1 — Prerequisites & Quick Setup

4 — How Admin Controls User Data

Access Control Flow

User registers → Gets "user" role by default
       ↓
Admin logs in → Sees all users in Admin Panel
       ↓
Admin clicks user → Opens User Modal
       ↓
Admin can:
  ✅ View full profile & search history
  ✅ Change role (user ↔ admin)
  ✅ Suspend account (blocks login)  
  ✅ Delete account permanently
  ✅ Monitor search patterns & behavior
       ↓
All actions happen instantly via secure API
(JWT authentication + admin-role verification)

Security Layers

LayerProtection
JWT TokenEvery API request must carry a valid access token
Role CheckAdmin endpoints verify user.role === 'admin' server-side
Frontend GuardAdmin page redirects non-admin users automatically
Password Hashingbcrypt with 12 salt rounds — passwords never stored in plain text
Token RefreshAuto-refresh system with 30-day refresh tokens

Data the Admin Has Access To

👥 User Accounts

Name, email, role, join date, last login, verified status, preferences

🔍 Search Activity

Every search query, brand, product, timestamp — per user & aggregated

📊 Platform Analytics

Trending brands, search volume, unique users per brand, time-based patterns

🔔 Alert System

Brand alert thresholds, registered brands, notification processing

6 — Client Talking Points

Key messages to emphasize during the demo:
  1. "Two-level access system" — Regular users see their own dashboard; admins see everything. Clean separation of concerns.
  2. "Full user lifecycle control" — Admin can create, view, edit, suspend, and delete any user account from one panel.
  3. "Real-time analytics" — The admin panel shows live trending data, search patterns, and platform health without any extra tools.
  4. "Built-in security" — JWT tokens, password hashing, role-based guards on both frontend AND backend. No shortcuts.
  5. "Scalable architecture" — MongoDB for data, Express for API, React for UI — industry-standard stack that scales.
  6. "Brand intelligence" — When enough users search for a brand, the system automatically detects the trend and can notify the brand. This is monetizable data.
  7. "Self-service for users" — Users manage their own profile, password, preferences, and can delete their account (GDPR-friendly).
  8. "Extension integration" — The browser extension feeds data back to this dashboard. Users install the extension, search for coupons, and all activity flows into the analytics engine.
